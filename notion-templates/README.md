# OSOP Notion Templates

10 workflow templates designed for Notion databases. Each `.osop` file maps directly to a Notion database schema where every node becomes a row with Status, Type, and Description properties.

## How to Use

1. Create a Notion database with columns: **Name** (title), **Type** (select), **Status** (select), **Description** (text), **Dependencies** (text)
2. Import the workflow steps from any `.osop` file below
3. Use Notion's Board view to track workflow progress

## Templates

| Template | Nodes | Use Case |
|----------|-------|----------|
| project-management.osop | 5 | Product lifecycle: idea → spec → dev → review → ship |
| content-calendar.osop | 5 | Content pipeline: ideate → write → edit → design → publish |
| hiring-pipeline.osop | 5 | Recruiting: source → screen → interview → offer → onboard |
| product-launch.osop | 5 | Launch: research → build → beta → marketing → launch |
| meeting-notes.osop | 5 | Meeting: agenda → notes → action items → follow-up → close |
| okr-tracking.osop | 4 | OKR: set objectives → define KRs → weekly check-in → review |
| customer-feedback.osop | 5 | Feedback: collect → categorize → prioritize → implement → notify |
| incident-management.osop | 5 | Incident: detect → triage → investigate → resolve → postmortem |
| design-review.osop | 5 | Design: brief → explore → prototype → review → approve |
| knowledge-base.osop | 5 | KB: draft → peer review → format → publish → maintain |
